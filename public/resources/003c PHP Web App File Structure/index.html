PHP Web App File Structure

Below is an example file structure for a PHP event booking system app. Like other technology, Ruby on Rails and Flask (Python), the structure encourages a Model View Controller design pattern. Informed by OOP, the structure encourages code separation and modularisation, data abstraction and encapsulation. It may take some getting used to, but it is a very common pattern

More information: Research / Investigate MVC in WebApps.

app/
  controllers/
    EventController.php
    BookingController.php
  models/
    Event.php
    Booking.php
  views/
    events/
      index.php
      create.php
      edit.php
      show.php
    bookings/
      index.php
      create.php
      edit.php
      show.php
config/
  database.php
public/
  css/
    styles.css
  js/
    main.js
  images/
    logo.png
routes/
  web.php
vendor/
  • The app/ directory contains the main application code.
  • The controllers/ directory contains the PHP controller files.
  • The models/ directory contains the PHP model files.
  • The views/ directory contains the HTML view files, organised by controller and action.
  • The config/ directory contains configuration files, such as the database configuration file.
  • The public/ directory contains static files, such as CSS, JavaScript, and image files.
  • The routes/ directory contains the application routes file.
  • The vendor/ directory contains the third-party PHP dependencies.

Please adjust this example based on your specific needs and preferences.
